# 分阶段实现规范

## 目标
只实现当前阶段范围内的变更，不跨阶段、不扩需求。

## 核心原则
- **胶水编程优先**：优先复用系统/标准库/现有代码。
- **最小改动**：只改必须文件与最小范围。
- **不引入新依赖**：若确需第三方，必须先说明理由并征得同意。
- **架构一致性**：遵守 `memory-bank/architecture.md`。

## 输出要求
- **Scope**：说明阶段边界与要改的文件。
- **Implementation**：按步骤说明修改点（必要时附最小 diff）。
- **Verification**：阶段内验收点与预期日志。
- **Assumptions**：缺信息时的最安全默认假设。

## 约束
- 不得引入跨阶段功能。
- 不得修改与本阶段无关的文件。
